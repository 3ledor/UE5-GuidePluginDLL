Unreal Engine 5 – Creating and Building a Plugin with Dynamic-Link Libraries (DLLs)

Who is this for?

This guide is for Unreal Engine 5 beginners or game developers who are trying to find their way into Unreal Engine 5. Basic knowledge of DLLs, programming and command shells is required because I won’t go into all of that.

And of course, you need the Unreal Engine 5.x installed with a compatible IDE.

What is this guide about?

In this guide I will explain how to create an Unreal Engine 5 plugin and how to build/package it. In addition, I will show how to use DLLs to extend and simplify your code or even use your own DLL files.

Why should I create and use plugins?

The Unreal Engine is built for plugins. It enables you to create a modular project that uses all the different plugins in a single eco-system. This will help you organize your code and split different features into different modules.

How to create a plugin in Unreal Engine 5?

The creation of the plugin will happen inside the project. So, it’s up to you if you want to use an already existing project or create a completely new one. To have a clean plate I would suggest creating a new one.

So, the first thing we need to do is create a new project. It does not matter what template you will use but to keep it small and clean I will use a “Blank” project. The only thing I would suggest is selecting “C++” so that it already creates the source files.

The next step is to open the “Plugins” Tool. To do this you have to select “Edit” in the top left of the editor and then select “Plugins”.

In this window you can add and remove plugins from your project. To create a new plugin, just click on the “Add” button on the top left. This will open the “New Plugin” window.

Here you can select from a few templates that will help you with some example functions. For my case I will select “Blueprint Library” because I want my plugin to create new blueprints to use.

You also must set the name of the plugin and if you want to, you can already set some additional information like “Author” or an “Description”. After selecting the template and setting the required information you can click on “Create Plugin” in the bottom right.

Your own plugin will automatically be created and enabled. In the content drawer of the project, you can already see the C++ file directories that were created.

You can know open the project in your IDE and there you will find a “Plugin” folder that contains another folder with the name of the plugin you just created. In this folder are all the files regarding your own plugin.

For me the “xxBPLibrary.cpp” and the “xxBPLibrary.h” files are important, because I want to create new blueprints.

In the header file of the blueprint library, you will already find an example function. As you can see from the example, you can also set the description of the plugin and some keyword for the search engine to find your blueprint in the editor.

To test my plugin, I will create my own function that will be available as a blueprint. Like in any normal C++ code you will declare the function in the header file and create the definition in the CPP file.

In my case I’ve created a function with the name “ThisWillReturnAString” that takes an integer and does completely nothing with it. In return my function does return a FString that I declared and defined inside of the function.

After saving and compiling the code, I can already check out my new blueprints in the editor of any actor or other object.

If your own function does not appear, don’t worry! Sometimes it’s necessary to restart the unreal engine or to even re-generate the project files. Simply right click on the uproject file in the file explorer and click on “generate visual studio project files”.

That’s already it! You’ve created your first plugin and even created a completely new blueprint!

How to build/package your own plugin?

There are two ways to build/package your previously created plugin. One is trough the unreal engine editor and the other one is using the unreal automation tools manually.

Let’s first start with the easy way through the editor!

Open the “Plugins” window and search for your own plugin.

Here you can change some information with “Edit” or build/package your plugin with “Package”. Just click on “Package” and select the destination for your plugin. After the compilation and package process was successful, you are already done!

In the file explorer you will find a file structure that looks like a normal unreal engine project. The only thing that we are interested in, is the “Plugins” folder. This folder does contain our packaged plugin.

The second and a bit more advanced way would be to use the “Unreal Automation Tool” or short “UAT” to build/package the plugin. The advantage of this way is that you will have a few more settings that you can choose from.

The “UAT” can be used directly through CMD (Command Shell) or with a “Batch file (.bat)” file. In my case I was using a .bat file to execute the build/package process.

At first you need to find the location of your unreal engine 5 installation and then find the “RunUAT.bat” file. In my case I found the file in the following directory “C:\Program Files\Epic Games\UE_5.4\Engine\Build\BatchFiles”.

With the known location of the “RunUAT.bat” file you can now create a batch file that executes this file with additional parameters.

The main parameter is the action that should be executed. In our case its “BuildPlugin”.

After that you must define the location of the “.uplugin” file of your plugin to the “-Plugin” parameter. You can find this file in the plugin folder of your unreal engine project. After you define the source of your plugin you must also define the destination to the “-Package” parameter. Here you can choose any location you want the plugin to be built/packaged in.

To get a list of additional settings you could add to your batch file, just run the “RunUAT.bat” file with “BuildPlugin -Help”. This will list all the available settings that you could additionally use.

That’s it! If you now run the batch file, your plugin will be built/packaged to the defined location!

How do I use DLLs in my plugin?

Implementing DLL files into your plugin is actually pretty simple with some help!

First, I will create a new folder inside of my plugin that will contain the DLL files. In my case I’ve created a folder with the name “DLL” that contains an “Include” and a “Lib” folder.

In the “include” folder I will paste all the headers of my DLL files. On the other hand, in the “lib” folder I will paste all the .dll and .lib files of my DLLs.

With that done, we will now have to make some changes to the “.Build.cs” file of our plugin. This file can be found in the source folder of our own plugin.

The first thing we will add is the path to our additional include folder we have just created.

The second thing we will add is the path to our additional libraries and set the dll to load.

With that we can already start using the DLL as “Include” in our plugin!

Sadly, adding the path of the DLL files is not all…

If you want to successfully build your project, you also must paste the .dll files to the binary folder of the unreal project.  You can either manually copy/paste the files to the binary folder or you can create a “PreBuildStep” to automatically copy the files to the right directory.

This can be done by editing the “.uplugin” file of the plugin and adding a copy command with the source location and target location.
